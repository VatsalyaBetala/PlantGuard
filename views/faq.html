FAQ

Frequently Asked Questions

Answers to the things people ask most about PlantGuard's AI-powered plant disease detection.

How do I upload images for analysis?
Go to the Upload page, drag a leaf photo (or click "Choose Files") into the dropzone, and hit Start AI Analysis. You can queue multiple images at once — results appear in the Library tab.
What image formats are supported?
JPEG, PNG, and WEBP up to 10 MB per file. For best accuracy, aim for well-lit close-ups of a single leaf filling most of the frame.
Where does my data go?
Nowhere, by default. Uploaded images live in the local uploads/ folder on the machine running PlantGuard. The only outbound call is the optional GPT-4o diagnosis explanation — and that only fires when you click "AI Analysis" on a specific image.
How does live recording work?
The Live page opens your webcam, samples a frame per second, and runs each frame through the same pipeline as a file upload. Diagnoses stream into your Library as frames finish processing.
How accurate is the model?
On the held-out PlantVillage test split we measure ~92% validation accuracy for the plant head and comparable or better for the per-plant disease heads. Out-of-distribution field photos will typically do worse — always verify high-stakes diagnoses with a human pathologist.
What is the Grad-CAM heatmap?
Grad-CAM is a gradient-based explainability technique that highlights which regions of the leaf most influenced the model's prediction. We target the last convolutional block of ResNet-50 (layer4[-1]) and overlay the resulting heatmap on the original leaf.
Is PlantGuard open source?
Yes — MIT licensed. The training pipeline, inference code, and this entire UI live in the GitHub repository.
Does it work on mobile?
The UI is fully responsive and the live-camera feature uses the browser's getUserMedia API, so a phone pointed at a leaf works as well as a laptop webcam. Run the backend on any machine the phone can reach over the network.
What do I get back from an analysis?
For each image: predicted plant species, predicted disease, a severity tag, a Grad-CAM heatmap, and an optional GPT-generated treatment write-up (symptoms, causes, prevention, remedies).
Can I run it offline on a Raspberry Pi?
Yes. Set MODEL_NAME=cnn_resnet50 and the pipeline runs on CPU. On a Pi 4/5 you'll get multi-second inference per image rather than sub-second GPU latency, but it works. Skip the LLM-explain feature if you want zero outbound calls.
